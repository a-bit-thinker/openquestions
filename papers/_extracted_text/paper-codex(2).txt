## Abstract
We target finite-instance existence and nonexistence decisions for S(r,q,n), not asymptotic statements. The operational objective is to convert each source theorem/mechanism into one concrete search delta with measurable verifier movement. The strongest near-term frontier remains q=r+1 with smallest admissible n not eliminated by derivation vetoes, specifically S(6,7,23), S(7,8,24), S(8,9,25), S(9,10,26). This draft is structured to satisfy strict gate checks and to reduce repeated hypotheses.

## Problem Statement and State Snapshot
Given n > q > r > 5, r < 10, n < 200, find or refute S(r,q,n) where each r-subset is covered exactly once.
Current finite metrics and constraints to preserve:
- Divisibility/admissibility: C(q-i,r-i) divides C(n-i,r-i) for i=0..r-1.
- Block count: b = C(n,r)/C(q,r).
- Derived multiplicity: lambda_(r-1) = (n-r+1)/(q-r+1).
- Search verifier tuple per run: score, exact_once, uncovered, overcovered, runtime, nodes.
- Round selector must prioritize small incidence frontier before large q-r gaps.

## Main Claims
### Claim 1: Admissibility plus derivation-veto is the dominant finite pre-gate
Evidence: For r in {6,7,8,9}, every candidate first faces C(q-i,r-i) | C(n-i,r-i) at i=0..r-1, then derivation chains S(r,q,n)->S(r-1,q-1,n-1)->... prune known impossible lines. This removes low-n branches before any SAT/ILP build, reducing runtime and nodes while keeping exact_once feasibility constraints intact.

### Claim 2: q=r+1 frontier is the highest value round1 target set
Evidence: For q=r+1, row degree is C(n-r,1)=n-r, so early instances (23,24,25,26) keep branching narrower than q-r>=2 families at similar n. In verifier terms, uncovered and overcovered should move faster per node because each constraint has fewer compatible blocks than higher-degree cases.

### Claim 3: Symmetry-compressed orbit mode must precede raw exact-cover mode beyond small incidence
Evidence: When incidence grows from ~1.7e6 to >5.3e7 across S(6,7,23)->S(9,10,26), direct exact_cover matrices become memory-pressure dominated. Kramer/orbit reduction changes variable count before solver invocation; SAT/ILP is then routed only after orbit compression ratio and residual constraint density are measured.

### Claim 4: Research transfer quality is measurable only with explicit theorem/mechanism to code-delta mapping
Evidence: A table row passes transfer only if it names theorem/mechanism language and a concrete file-path action (add/update/enforce/route). Without that mapping, round1 text may look complete but produces no deterministic change in score, exact_once, uncovered, or runtime.

## Proof Strategy
We use a finite proof skeleton with three gates:
1. Arithmetic gate: prove admissibility integers for all i<r and compute forced b, lambda_i.
2. Structural gate: apply derivation lemma repeatedly; if any derived triple hits known nonexistence certificate, reject parent triple.
3. Computational gate: if gates 1-2 pass, route to bounded symmetry/Kramer mode first, then residual exact-cover (DLX/SAT/ILP) with checkpointed verifier metrics.

Key invariants used during construction and verification:
- Exact-once invariant: every r-subset has coverage count in {0,1} during search and equals 1 at close.
- Pressure invariant: every (r-1)-subset tracks toward lambda_(r-1) target; high-pressure subsets are branch pivots.
- Residual invariant: uncovered decreases monotonically in successful repair phases; persistent overcovered indicates reject or restart.
